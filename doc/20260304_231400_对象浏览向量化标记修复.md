# 对象浏览向量化标记修复（2026-03-04 23:14）

## 背景
- 需求：每个表或视图都需要展示向量化状态标记，便于用户直接识别对象的向量化状态。
- 现状问题：`App.vue` 出现了编码损坏导致的模板/脚本语法错误，前端无法通过类型检查与构建。

## 本次改动
1. 修复 `apps/desktop/src/App.vue` 语法与模板损坏问题
- 修复了大量断裂字符串、未闭合属性、缺失结束标签，恢复 `App.vue` 可编译状态。
- 修复对象浏览、连接配置、图表操作等区域的错误文案占位，避免模板表达式解析失败。

2. 落地“表/视图向量化状态标记”
- `objectRows` 计算逻辑中，`tables` 继续显示数据库向量化状态。
- 新增 `views` 分支：视图列表也读取并显示当前数据库向量化状态（`vectorizeStatus/vectorizeMessage/vectorizeUpdatedAt`）。
- 其余对象类型保留 `NOT_VECTORIZED` 逻辑。

3. 补齐上下文菜单打开查询页签时的对象结构
- 在 `triggerContextAction('queryData')` 里构造 `ObjectRow` 时补充向量化字段，满足类型约束并保证页签内状态信息完整。
