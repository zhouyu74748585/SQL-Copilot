# Review Todo 审查执行

## 执行目标

在收到 review 请求时，基于 `docs/review` 最新待办记录执行一次完整代码审查，并将审查结果落盘归档。

## 执行步骤

1. 运行当前技能目录中 `scripts/find_latest_review_todo.sh` 获取最新待审文档绝对路径。
2. 读取该文档中的“需求描述”和“变动文件”。
3. 仅针对“变动文件”进行审查，必要时结合上下文文件。
4. 按“审查清单”生成结果，输出必须包含：
   - 问题清单（按类别分组，含位置参考）
   - 修复建议（每个问题给出可执行建议或示例）
   - 风险等级（高/中/低）
5. 将审查结果写入临时 markdown 文件。
6. 运行 `scripts/complete_review_todo.sh --result-file <临时文件路径> [--todo-file <待审文档路径>]`：
   - 将审查结果追加到待审文档
   - 重命名为 `reviewDone-{yyyyMMddHHmmss}.md`
7. 最终回复中返回生成的 `reviewDone` 文件绝对路径。

## 审查清单

1. 是否完整实现需求预期行为。
2. 是否存在未覆盖逻辑分支。
3. 在边缘或异常条件下是否表现正确。
4. 是否存在逻辑错误、未处理异常或边界问题。
5. 是否有安全风险（输入未验证、权限检查缺失、注入风险等）。
6. 是否符合编程规范与最佳实践（命名、结构、必要注释、可读性）。
7. 是否有性能问题或可优化点（重复计算、不必要循环、N+1查询等）。

## 输出格式

使用如下结构写入审查结果：

```markdown
### 问题清单
#### <类别>
- [风险: 高/中/低] 问题描述（位置：文件路径:行号 或 函数名）

### 修复建议
- 对应问题的修复方案或示例代码

### 结论
- 总体风险：高/中/低
```

## 注意事项

- 全过程使用中文。
- 仅处理 `reviewTodo-*.md`，不处理 `reviewDone-*.md`。
- 若不存在待审文件，明确说明并停止。
