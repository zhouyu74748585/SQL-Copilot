# SQL Copilot 产品需求文档（PRD）

## 1. 产品概述

SQL Copilot 是一款基于 Spring Boot + GraalVM 原生编译 + Electron
架构开发的 AI 原生数据库管理工具。

产品目标不是简单的 SQL
生成工具，而是一个具备结构感知能力、执行闭环能力、风险控制能力和上下文记忆能力的智能数据库工作台。

------------------------------------------------------------------------

## 2. 用户需求与功能列表

### 2.1 数据库连接与管理

-   多数据库支持（MySQL、PostgreSQL、SQL Server、SQLite、Oracle 等）
-   多连接管理
-   SSH 隧道支持
-   只读连接模式
-   多环境标识（开发 / 测试 / 生产）
-   生产环境默认只读
-   高风险 SQL 二次确认

------------------------------------------------------------------------

### 2.2 Schema 结构管理模块

-   自动读取数据库 schema
-   表结构、索引、外键关系加载
-   表大小统计与行数估算
-   ER 图生成
-   表依赖图
-   自动生成字段业务说明
-   自动识别冗余字段与弱索引

------------------------------------------------------------------------

### 2.3 AI 查询工作流（核心模块）

工作流程：

用户输入需求\
↓\
AI 生成 SQL\
↓\
自动 EXPLAIN 分析\
↓\
风险评估\
↓\
执行 SQL\
↓\
异常自动修正

能力： - 自动修正字段错误 - 自动补充 JOIN 条件 - 自动 LIMIT 控制 -
自动分页优化

------------------------------------------------------------------------

### 2.4 SQL 风险分析引擎

-   全表扫描检测
-   未使用索引检测
-   update/delete 无 where 检测
-   锁表风险分析
-   大事务风险提示

------------------------------------------------------------------------

### 2.5 数据洞察模块

-   大表排名
-   冗余索引检测
-   慢查询分析
-   表增长趋势分析
-   索引优化建议
-   分区建议

------------------------------------------------------------------------

### 2.6 AI 数据分析模式

-   自然语言生成聚合 SQL
-   自动绘图（折线图、柱状图、饼图）
-   趋势分析说明
-   数据预测能力

------------------------------------------------------------------------

### 2.7 DDL & 迁移辅助

-   Schema diff 对比
-   自动生成迁移脚本
-   自动生成回滚脚本
-   分区改造建议

------------------------------------------------------------------------

### 2.8 查询编辑器功能

-   SQL 高亮
-   自动补全
-   结构感知补全
-   多标签页支持
-   查询结果导出（CSV / Excel / JSON）

------------------------------------------------------------------------

## 3. 技术栈选型

### 桌面端

-   Electron
-   Vue3
-   Ant Design Vue
-   Vite
-   样式主体参考navicat

### 后端

-   Spring Boot
-   GraalVM Native
-   SQLite（本地元数据）
-   JDBC 统一连接层

### AI 集成

-   OpenAI API（Tool Calling 模式）
-   自定义 Agent Loop
-   Schema Context Builder
-   Token 预算控制模块
-   上下文记忆模块
-   支持本地与线上模型向量化
-   向量化及向量化查询优化，对sql历史进行更合理的处理以便能够命中
------------------------------------------------------------------------

## 4. 风险与挑战

-   AI Token 成本控制
-   SQL 安全与误操作风险
-   多数据库方言兼容
-   Explain 格式差异
-   大表性能问题

------------------------------------------------------------------------

## 5. 成功标准

-   AI 生成 SQL 准确率 ≥ 85%
-   风险检测准确率 ≥ 95%
-   查询响应时间 ≤ 2 秒（常规查询）
-   UI 操作流畅

------------------------------------------------------------------------

## 6. 智能ER图功能实现说明（2026-03-04）

- 对象浏览页操作栏左侧新增“智能ER图”入口。
- 用户可在当前库内多选目标表（最多30张），确认后打开工作台 ER 新 Tab。
- 新增后端接口：`POST /api/schema/er/graph`。
- ER 图关系分两类：
  - 外键关系：实线
  - AI 推断关系：虚线（默认置信度阈值 0.6）
- 首次打开 ER Tab 自动触发 AI 推断；支持“重新选表”和“刷新AI关系”。
- AI 推断失败时不影响主流程，页面仍展示外键关系与表节点。
- 图渲染采用 ECharts Graph 组件。
