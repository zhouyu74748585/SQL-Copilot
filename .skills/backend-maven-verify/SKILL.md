# 后端 Maven 打包验证

## 概述
用于后端任务完成后的强制构建验证：只有 Maven 打包成功且无报错才算任务完成。

## 工作流程
1. **判定是否触发**：只要涉及后端代码/配置/资源改动，或用户要求“验证/打包/构建”，就必须执行 Maven 打包。
2. **选择命令**：优先采用项目约定的 Maven 打包命令；如未明确，使用 `mvn clean package`。
   - 多模块仓库：进入后端模块目录执行，或使用 `mvn -pl <backend-module> -am clean package`。
3. **执行并判定**：构建成功且无报错，任务才算完成；若失败，必须修复错误并重新执行，直到成功。
4. **回复说明**：明确说明已执行的命令与结果；若未成功，说明失败原因与尚未完成。

## 注意事项
- 若执行命令需要网络或更高权限，按运行环境要求请求授权后再执行。
- 避免“未运行但声称完成”；未成功构建不得标记任务完成。
