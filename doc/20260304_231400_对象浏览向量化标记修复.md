## 追加更新（2026-03-04 23:39）
- 目标：改造向量化（ONNX BGE-M3）推理链路，优先调用本地 GPU，覆盖 Windows 显卡与 macOS M 系列显卡场景。

### 本次后端改动
1. `apps/server/src/main/java/com/sqlcopilot/studio/service/rag/impl/OnnxBgeM3EmbeddingServiceImpl.java`
- 新增 ONNX Execution Provider 选择逻辑，支持配置：
  - `AUTO`（默认）
  - `CPU`
  - `DIRECT_ML`
  - `CORE_ML`
- `AUTO` 策略：
  - Windows：优先 `DIRECT_ML`，不可用则回退 `CPU`
  - macOS + arm64/aarch64（M 系列）：优先 `CORE_ML`，不可用则回退 `CPU`
- 启动时读取 `OrtEnvironment.getAvailableProviders()`，仅在 provider 可用时启用；启用失败时写日志并自动回退。
- 初始化日志增加 provider 解析结果，便于定位本机是否成功走 GPU。

2. `apps/server/src/main/resources/application.yml`
- 新增默认配置：
  - `rag.embedding.execution-provider: AUTO`
  - `rag.embedding.directml-device-id: 0`

## 追加更新（2026-03-05 00:05）
- 问题：Windows 环境向量化仍未使用 GPU。

### 根因确认
1. 运行时 Provider 探针结果：
- 使用 `onnxruntime` 基础包时，`OrtEnvironment.getAvailableProviders()` 仅返回 `CPU/AZURE`，不包含 `DIRECT_ML/CUDA`。

2. 结论：
- 仅靠原有 `onnxruntime` 依赖无法在当前 Windows 环境命中 GPU Provider。

### 本次修复
1. `apps/server/pom.xml`
- 新增属性 `onnxruntime.artifact-id`，默认 `onnxruntime`。
- 新增 Windows 自动激活 profile：`windows-onnx-gpu`，将依赖切换为 `onnxruntime_gpu`。

2. `apps/server/src/main/java/com/sqlcopilot/studio/service/rag/impl/OnnxBgeM3EmbeddingServiceImpl.java`
- Provider 常量新增 `CUDA`。
- `AUTO` 策略在 Windows 下改为：`CUDA -> DIRECT_ML -> CPU`。
- 增加 `rag.embedding.cuda-device-id` 配置读取并在 `options.addCUDA(cudaDeviceId)` 启用 CUDA。

3. `apps/server/src/main/resources/application.yml`
- 新增 `rag.embedding.cuda-device-id: 0`。

### 验证结果
1. Provider 探针验证
- 使用 `onnxruntime_gpu-1.19.2.jar` 时，`getAvailableProviders()` 返回 `CPU/CUDA/TENSOR_RT`，确认 CUDA Provider 可见。

2. 构建与启动验证
- 后端 `mvn clean package`：通过。
- 后端 `mvn --% clean spring-boot:run -Dspring-boot.run.arguments=--server.port=18082`：成功监听（`18082`）。
- 前端 `npm run -w @sqlcopilot/desktop build -- --emptyOutDir`：通过。
- 前端 `npx vite preview --host 127.0.0.1 --port 4174`：成功监听（`4174`）。
- 验证后已停止本次拉起的进程。
