# 前端构建与类型检查验证

## 概述
在完成前端需求/改动后，必须通过一次完整的前端构建与类型检查验证，成功且无报错才算任务完成；若失败必须修复后重试。

## 触发条件
- 任何前端代码、样式、配置改动。
- 用户要求“打包/构建/验证/检查/发布前验证”等。

## 工作流程
1. **进入前端目录**：根据仓库约定（例如 `frontend/pc-web`）。
2. **安装依赖（如缺失）**：若 `node_modules` 缺失或依赖变更，运行 `npm install`（或项目约定的包管理器）。
3. **执行构建**：运行 `npm run build`（或约定命令，如 `npm run build:prod`）。
4. **执行类型检查**：运行 `npm run type-check`（若无该脚本，使用 `npm run build -- --type-check` 或项目等价命令；若仍缺失需在回复中说明并补充等效命令）。
5. **判定结果**：只有两步都成功且无报错才算验证通过；任何一步失败必须修复并重跑直至成功。
6. **回复说明**：明确写出所执行的目录与命令、结果；失败时给出错误摘要与后续修复计划，切勿标记为完成。

## 注意事项
- 遵循项目既有脚本优先，若脚本名不同需按项目实际调整并在回复中注明。
- 如命令需网络/权限，按环境要求先申请授权再执行。
- 禁止“未跑即通过”的描述；未通过不得视为完成。
