### 1: 当前工程需要实现 GraalVM 本地编译，以提高软件的启动速度和响应时间。在实现过程中需要考虑到相关动态代码、反射、动态类加载的配置和优化，确保软件在本地编译后的运行效果与原始版本相同。
### 2: 技术栈选型
- 前端框架 ： Vue3 + TypeScript (根据Vue生态选择，提供现代化UI组件)
- UI组件库 ：Element UI / Ant Design Vue (根据Vue生态选择，提供现代化UI组件)
- 打包工具 ：Webpack / Vite
- Electron框架 ：Electron (核心)
- 后端 ：后端使用 GraalVM 17 本地编译+Spring Boot技术栈
### 3: 每完成一个对话的开发工作，都需要在/docs目录中输出该阶段任务总结，文件名格式固定为 `yyyyMMddHHmmss-{topic}.md`（UTF-8 编码，便于排序）。未进行文件修改的对话不需要记录。如果存在相同或相似主题文档，必须追加到该文档而非重复创建，保证同一主题仅保留一份持续记录。
### 4: 每一个新对话开始新工作之前，都需要回顾之前完成的工作总结，并优先读取 `/docs/prd.md`；如果不存在则读取 `/docs/SQL_Copilot_PRD.md`，根据产品需求进行开发
### 5: 每一个任务均以“启动验证”（后端启动+前端预览）作为结束标准。如果不能成功启动，需要及时修复错误，直到能成功启动为止。每次启动应该clean一次，避免之前生成的文件对是否成功编译的判断产生影响
### 6: 除非明确要求，不需要兼容旧数据
### 7：开发可能在不同的环境中进行，需要考虑到不同环境下文本的编码避免乱码产生，文本统一使用UTF-8编码
### 8：每次进入新工程时，若项目根目录不存在`.skills`目录，则创建该目录并从`/Users/zhouyu/.codex/skills/`复制已安装的skills（排除`.system/skill-creator`与`.system/skill-installer`）；后续任务优先使用项目内`.skills`中的技能定义
### 9：若用户显式指定技能（如`$skill-name`）或任务与某技能领域匹配，必须读取并遵循对应`.skills/<skill>/SKILL.md`
### 10：若项目内技能缺失或不可读，需要简要说明原因，并采用可行的降级方案继续执行
### 11：生成或维护`AGENTS.md`时，内容默认使用中文
### 12：与用户交互时默认使用中文回复，除非用户明确要求使用其他语言

### 技能引导规则（由 project-skill-bootstrap 维护）
- 优先使用`<project-root>/.skills`中的技能。
- 若新工程缺少`.skills`，则创建并从`/Users/zhouyu/.codex/skills/`复制技能（排除`.system/skill-creator`与`.system/skill-installer`）。
- 若用户显式指定技能（如`$skill-name`）或任务命中技能领域，必须读取对应`.skills/<skill>/SKILL.md`。
- 文本编码默认使用UTF-8。
- 除非用户明确要求，否则不兼容旧数据。
- 生成或维护`AGENTS.md`时默认使用中文。
- 与用户交互时默认使用中文回复，除非用户明确要求其他语言。
